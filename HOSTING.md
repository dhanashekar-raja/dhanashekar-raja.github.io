# Hosting This Portfolio

This site is ready to host as a static website.

## Recommended option: GitHub Pages

Why this is a good fit:

- Free for static portfolio sites
- Easy to connect with your GitHub profile
- No build setup required
- Works well with the current file structure

## Files already ready for GitHub Pages

- `index.html`
- `styles.css`
- `script.js`
- `assets/`
- `.nojekyll`

## How to publish on GitHub Pages

1. Create a new GitHub repository, for example `dhanashekar-raja.github.io` or `portfolio`.
2. Upload all files from this folder to the repository root.
3. In GitHub, open `Settings` > `Pages`.
4. Under `Build and deployment`, choose:
   - `Source`: `Deploy from a branch`
   - `Branch`: `main`
   - `Folder`: `/ (root)`
5. Save the settings.
6. Wait a few minutes for GitHub Pages to publish the site.

If you use a repository named `dhanashekar-raja.github.io`, the site URL will usually be:

- `https://dhanashekar-raja.github.io/`

If you use a repository named `portfolio`, the site URL will usually be:

- `https://dhanashekar-raja.github.io/portfolio/`

## Alternative option: Netlify

Netlify is also a good option if you want:

- easier drag-and-drop publishing
- custom domain setup later
- simple continuous deployment from GitHub

For Netlify:

1. Sign in to Netlify.
2. Drag this whole folder into the Netlify dashboard, or connect the GitHub repository.
3. No build command is needed.
4. Publish the site as a static project.

## Suggested next step

Use GitHub Pages first. It is the simplest and most aligned with a personal portfolio.
